# 要件定義書

## 1. 目的
- 罫線あり表を含むPDFから、崩れを最小化したMarkdownテーブルを生成する。
- 既存の画像ラスタライズ中心方式ではなく、ベクトル情報+レイキャストのハイブリッドで精度を高める。

## 2. スコープ
- 対象: Node.js環境でのPDF→Markdown変換。
- 対象: 罫線あり表（完全格子または軽微な欠損罫線）。
- 非対象: 罫線なし表、OCR前提のスキャン画像PDF。

## 3. 利用ライブラリ
- `@typescript/native-preview`
- `oxlint`
- `typescript`
- `vite`
- `vitest`

## 4. 機能要件
- PDFファイルを入力し、ページ単位で表を抽出する。
- ベクトル罫線抽出とテキスト起点レイキャストを統合してセル推定する。
- Markdownテーブルとして出力する。
- CLIとライブラリAPIの双方を最小提供する。
- デバッグ用に中間テーブルJSONを出力可能にする。

## 5. 非機能要件
- 再現性: 同一入力に対して同一出力を返す。
- 保守性: TypeScript strict modeで型安全に保つ。
- 品質: `oxlint` と `vitest` を開発ゲートにする。
